V.II.I. Sortie vidéo du système

La sortie vidéo concerne généralement l’utilisation d’une carte graphique.

Il est essentiel de configurer la sortie vidéo correctement faute de quoi l’écran demeurera noir.

Pour les cartes graphiques Nvidia, veuillez suivre les instructions figurant à la page des réglages Nvidia. Réglages Nvidia.

Pour les cartes graphiques ATI, veuillez suivre les instructions figurant à la page des réglages ATI.Réglages ATI

V.II.II. Entrée vidéo du système Entrée vidéo du système

Scenic peut effectuer la capture vidéo via une carte de capture vidéo v4l2, une caméra IIDC ou DV (également appelée iLink). Ces dernières ont une latence importante (souvent entre 150 et 250ms) et ne devraient pas servir comme entrée vidéo si une faible latence est désirée.

Video4Linux

Connectez la caméra. Pour ce faire. Veuillez vous référer à notre page relative à l’installation matérielle. Installastion matérielle

Il s’agit maintenant de modifier vos réglages vidéo en fonction de vos besoins spécifiques.

Entrée vidéo Firewire (DV ou IIDC)

* L’usager doit être membre des groupes « disk » et « vidéo ». Dans le cas contraire, l’administrateur système doit inclure l’usager au groupe « disk » (tel qu’indiqué plus haut).

Sous Karmic Koala, il vous faut préciser l’attribution des équipements pour raw1394. Créez une règle udev pour changer le groupe de l’équipement /dev/raw1394:

echo 'KERNEL=="raw1394", NAME="raw1394", GROUP="disk"' | sudo tee -a /etc/udev/rules.d/99-raw1394.rules

sudo /etc/init.d/udev restart

De plus, sous toutes les distributions, il est conseillé de charger le module raw1394 même si aucune caméra dc1394 n’est utilisée pour que les sollicitations de libdc (ex. : milhouse --list-cameras) ne génèrent pas de message d’erreur. Pour ce faire, ajoutez raw1394 aux fichiers des modules /etc/:

sudo modprobe raw1394

echo "raw1394" | sudo tee -a /etc/modules

Se référer à ce rapport de bogue et cette documentation pour de plus amples renseignements.

Vous pouvez tester la caméra DV avec :

gst-launch dv1394src ! dvdemux ! dvdec ! xvimagesink

Vous pouvez tester la caméra IIDC avec :

gst-launch dc1394src ! ffmpegcolorspace ! xvimagesink

V.II.III. Test vidéo

Dans Scenic, il est facile de prévisionner les entrées vidéo des équipements video4Linux2 dans l’onglet « Video ». Les autres pilotes d’entrée ne sont pas encore supportés par Scenic mais le sont par milhouse :

Choisissez la source vidéo en fonction du tableau suivant :

v4l2src Équipements video4linux2
v4lsrc Équipements video4Linux (v1) – uniquement supportés par ligne de commande
dv139src Caméras et lecteurs DV – uniquement supportés par ligne de commande
dc1394src Caméras IIDC 1.31– uniquement supportés par ligne de commande

milhouse --localvideo --videosource v4l2src

Vous pouvez ajouter de nombreuses options telles que :

  • --aspect-ratio (format)
  • --width (largeur)
  • --height (hauteur)
  • --fullscreen (plein écran)
  • --flip-video
