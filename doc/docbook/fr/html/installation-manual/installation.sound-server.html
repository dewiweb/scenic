V.IV.I. Configuration de jackd

Démarrez le serveur audio jackd avec qjackctl situé dans « Applications/Sound & Video/Jack Control »

Cliquez sur le bouton « setup ».

Voici quelques exemples de configuration en temps réel pour qjackctl :

Parameter Value
Realtime Yes
Priority Default
Frames/Period 64
Sample Rate 48000
Periods/Buffer 2
Port Maximum 256
Timeout (msec) 500
Interface Default
Audio Duplex freebob/ffa
Input Latency Default
Output Latency Default
Latency 2.67 msec

Notre page dédiée aux réglages de qjackctl vous indique comment d’abord ajuster les paramètres en fonction de votre carte son spécifique. Ces paramètres sont précisés dans le tableau de réglages de JACK ci-dessous.

Veuillez noter que ces réglages dépendent également des spécifications de votre ordinateur et peuvent varier d’un type d’installation à l’autre.

Voici un tableau de l’ensemble des cartes son testées avec les réglages de JACK appropriés :

Carte de son -d backen -r taux d'échantillonage - p images/période -n pérides/tampon Latence
MaudioDelta 1010 (regular or LT) -d alsa -r 48000 -p 64 -n 2 2.67 msec
EdirolFirewire Fa101 -d freebob -r 48000 -p 64 -n 2 2.67 msec
Intel HDAudio -d alsa -r 48000 -p 512 -n 4 16 msec

-v = verbose, -R = mode temps réel, -d alsa règle le backend à ALSA (utiliser plutôt -dfreebob pourles cartes firewire), -r 48000 règle le taux d’échantillonnage à 48000 échantillons/sec, -p64 règle le nombre d’images/période à 64, and -n2 règle le nombre de périodes/tampon à 2.

Si vous utilisez ALSA, vous rencontrerez potentiellement un problème dans la séquence de chargement du module. Pour y remédier, utilisez plutôt les noms ALSA plutôt que l’index (hw:0) pour spécifier le type de carte. Voici comment :

Pour sélectionner la carte son appropriée, vérifiez tout d’abord le nom de la carte son à utiliser avec aplay.

Voici un exemple de sortie avec aplayaplay -l output:

card 0: Intel [HDA Intel], device 0: ALC889A Analog [ALC889A Analog]
  Subdevices: 1/1
  Subdevice #0: subdevice #0
card 0: Intel [HDA Intel], device 1: ALC889A Digital [ALC889A Digital]
  Subdevices: 1/1
  Subdevice #0: subdevice #0

Copiez le nom de la carte son (ex. : « HDA Intel ») et collez le dans le fichier .asoundrc localisé dan svotre répertoire $HOME. Ex. : Pour utiliser la carte son M2496 :

ex:

gedit .asoundrc

Fichier .asoundrc avec le nom m2496 ajouté. 

pcm.m2496 {
   type hw;
   card M2496;
}
ctl.m2496 { 
   type hw;
   card M2496;
}

Il ne vous reste qu’à ajouter le nom de la carte son dans le champ de l’interface de qjackctl. Par exemple, la m2496:

  • Cliquez sur l’onglet « Options ».

Vérifiez que seules les options suivantes sont sélectionnées :

Paramètre Valeur
Capture Standard Output Yes
XRUN detection regex xrun d'un minimum de ([0-9|\.]+) msecs
  • Cliquez sur l’onglet « Misc ».

Vérifiez que seules les options suivantes sont sélectionnées :

Paramètre Valeur
Start JACK audio server on application startup Yes
Save JACK audio server configuration to: .jackdrc
Confirm server shutdown Yes
Enable ALSA Sequencer support Yes

NOTE : En cas de message d’erreur, redémarrer l’ordinateur et essayez à nouveau.
