7.4. Milhouse Help

Type milhouse --help to have informations on the various available option 

milhouse --help

Help Options:

-?, --help

produce help

-r, --receiver

this process is a receiver

-s, --sender

this process is a sender

-i, --address

provide ip address of remote host address

-v, --videocodec

videocodec (mpeg4, h264, h263, theora)

-a, --audiocodec

audiocodec (raw, vorbis, mp3)

-k, --videosink

video output (xvimagesink, glimagesink, sharedvideosink, ximagesink)

-l, --audiosink

audio output (jackaudiosink, alsasink, pulsesink)

-t, --audioport

audioport number (1024-65535)

-p, --videoport

videoport number (1024-65535)

-f, --fullscreen

display video in fullscreen

-B, --shared-video-id

name for shared video buffer output (only applies to sharedvideosink)

-o, --deinterlace

deinterlace video (receiver side only)

-d, --videodevice

name of videodevice (/dev/video0, /dev/video1)

-q, --audiodevice

name of audiodevice (hw:0, hw:2, plughw:0, plughw:2)

--videolocation

video file location <filename> (only applies when videosource=filesrc)

--audiolocation

audio file location <filename> (only applies when audiosource=filesrc)

-n, --screen

xinerama screen num (0, 1)

-w, --version

outputs version number

-c, --numchannels

The numbers of audio channels to use.

-x, --videobitrate

videobitrate: The amount of video data being encoded; expressed in bits per second

-e, --audiosource

audio input (jackaudiosrc, alsasrc, pulsesrc)

-u, --videosource

video input (v4l2src, v4lsrc, dv1394src)

-z, --timeout

time in ms to wait before quitting, 0 means run indefinitely

-b, --audio-buffer-usec

length of receiver's audio buffer in microseconds, must be > 11333

-g, --jitterbuffer

length of receiver's rtp jitterbuffers in milliseconds, must be >= 1

-G, --camera-number

camera id for dc1394src (0,1)

-U, --camera-guid

camera guid for dc1394src

-I, --mulitcast-interface

interface to use for mutlicast (eth0, eth1)

-j, --enable-controls

enable gui controls for jitterbuffer

-J, --disable-jack-autoconnect

prevent milhouse's jack audio ports don't connect on startup

-O, --jack-client-name

name of jack-client

-C, --negotiate-caps

force rtp media capabilities (caps) to be communicated over tcp

-D, --debug

level of logging verbosity (critical=1, error=2, warning=3, info=4, debug=5, gst-debug=6)

-W, --window-title

title for video output window

-F, --framerate

framerate for video capture (15,30)

-H, --list-cameras

list connected cameras

-N, --width

width for video capture, must be specified for sender and receiver

-Y, --height

height for video capture, must be specified for sender and receiver

-P, --display-width

width for video on display (may scale input)

-Q, --display-height

height for video on display (may scale output)

-M, --grayscale

force dc1394 capture to grayscale

-A, --aspect-ratio

picture aspect ratio (4:3, 16:9)

--localvideo

display local video only
